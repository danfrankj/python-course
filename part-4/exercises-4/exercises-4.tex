\item{\textbf{Copys and Views}}
NumPy arrays are objects and follow the same assignment and copy rules as ordinary python objects. However, when we slice an array python returns a \textit{view} on that same data, meaning that a new object is created but shares the same underlying data. Integer indexing and boolean indexing do not create views.

\begin{enumerate}
\item
\begin{verbatim}
>>> A = np.ones((2, 5))
>>> B = A
>>> A[0,0] = 42.
>>> A[0, 0] == B[0, 0].
\end{verbatim}



\begin{verbatim}
>>> A = np.ones((2, 5))
>>> B = A[:, 1:3]
>>> B[0,0] = 42.
>>> A[0, 1] == 1.
\end{verbatim}

\item  

\begin{verbatim}
>>> A = np.ones((2, 5))
>>> B = A[:, np.array([False, True, True, False, False])]
>>> B[0, 0] = 42
>>> A[0, 1] == 1
\end{verbatim}


\item  

\begin{verbatim}
>>> A = np.ones((2, 5))
>>> B = A[:, np.array([1, 3])]
>>> B[0, 0] = 42
>>> A[0, 1] == 1
\end{verbatim}

\item  

\begin{verbatim}
>>> A = np.ones((2, 5))
>>> B = A[:, 1:3]
>>> A += 1
>>> np.all(B == 2)
\end{verbatim}
\end{enumerate}
